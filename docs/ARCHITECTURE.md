# Architecture

Native SwiftUI client for Mercury's `/api/v1`. Swift 6, default-MainActor isolation
([ADR 0003](adr/0003-swift6-default-mainactor.md)), zero third-party dependencies, project
generated by XcodeGen ([ADR 0002](adr/0002-xcodegen-with-gitignored-project.md)).

## Layers

```
Mercury/
├── App/        composition root and shell
│   MercuryApp → AppDependencies → RootView → MainTabView
├── Core/       transport-facing plumbing (no UI)
│   AppConfig, TokenStore, APIError, APIClient (+ MercuryAPI protocol), AuthService
├── Models/     Decodable DTOs mirroring the wire format
│   Track, Account, Dashboard, Vocab
└── Features/   one folder per screen family: View (+ ViewModel where stateful)
    Session/  Auth/  Onboarding/  Dashboard/  Vocab/  Profile/
```

Dependency direction is strictly downward: Features → Core/Models; Core knows nothing about
views. View models are `@Observable` classes owning a `(any MercuryAPI)`; views create them in
`init` via `@State`.

## Session state machine

`SessionModel` (Features/Session) is the single source of truth for the auth lifecycle.
`RootView` switches on its phase:

```
loading ──bootstrap──▶ signedOut ──sign-in/up──▶ onboardingRequired ──PUT settings──▶ ready
   │                        ▲                        (settings == nil)                  │
   └──▶ failed(msg) ─retry─┘◀───────────── any 401 (forceSignOut) ────────────────────┘
```

- Bootstrap: Keychain token? → `GET /me`; `settings == null` gates onboarding (mirrors the
  server's `onboarding_required` 403).
- Any 401 → `APIClient.onUnauthorized` → token purge → `signedOut`. Transport failures at
  bootstrap keep the token and show retry instead — a dead network must not log anyone out.

## Networking

Hand-written client ([ADR 0001](adr/0001-hand-written-client-over-openapi-generator.md)),
bearer-only and cookie-free ([ADR 0004](adr/0004-cookie-free-bearer-auth.md)). The `HTTPTransport`
protocol is the seam between clients and URLSession; tests inject a closure-backed
`StubTransport` and never touch the network.

Decoding rules worth knowing:

- Dates: custom ISO-8601 strategy with fractional seconds (`JSONCoding.makeDecoder()`) — the
  server's JS timestamps break Foundation's stock `.iso8601`.
- Open sets where the server can grow (`APIErrorCode`, `ScoreKind` → `.unknown`), closed enums
  where the contract is stable (`Track`, `Grade` = literal SM-2 values 1/3/4/5).
- Flattened server shapes (`VocabWord` + flags) decode via `init(from:)` composition
  (`OverviewWord`, `StudyCard`).
- Known error codes map to localized client copy; unknown codes surface the raw server message
  (`APIError.localizedMessage(for:)`).

## Testing strategy

- **Fixture-pinned DTO tests**: `MercuryTests/Fixtures/*.json` are captured from a live server by
  `scripts/refresh-fixtures.sh`; assertions target capture-deterministic invariants (fresh
  account, toeic onboarding, seeded content shape), never run-dependent values, so recapture
  can't break them.
- **View-model tests** drive state machines through `MockAPI` (closure-backed `MercuryAPI`).
- **One UI smoke** (`MercurySmoke` scheme, local-only): register → onboard → dashboard → flip
  and grade a card against a real backend. CI runs only the hermetic unit suite.

## Known seams (deliberate, revisit on trigger)

- **Debug server override needs a relaunch**: `AppDependencies` reads `AppConfig` once at
  launch; the Profile-tab override signs you out and applies on next start. Fixing it means
  rebuilding the dependency graph at runtime — not worth it for a debug convenience.
- **`MercuryAPI` is one protocol (8 methods)**: split per-feature when the drills phase grows the
  surface, not before.
- **Offline is deferred by design** — sketch in
  [ADR 0005](adr/0005-offline-study-deferred.md).
- **UI-test hooks** (`MERCURY_BASE_URL_OVERRIDE`, `MERCURY_RESET_SESSION`,
  `MERCURY_DISABLE_ANIMATIONS`) are read in DEBUG builds only.
